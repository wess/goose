Cargo for C · one binary · one goose.yaml

A package manager for C, inspired by Cargo.

Goose handles project scaffolding, dependency management, building, testing, and installation — all through a single command-line tool and a simple goose.yaml configuration file.

Features

Everything you reach for in Cargo — built for C.

Scaffold, depend, build, test, and ship from one tool and one manifest.

📦

Project scaffolding

goose new creates a ready-to-build project, complete with a goose.yaml, src/main.c, and .gitignore.

🔗

Dependency management

Add packages from any git repository or a local path. Transitive dependencies resolve recursively into packages/.

🗂️

Path dependencies

Use path: for local packages — like Cargo's path = "..." — ideal for monorepos and submodules.

⚙️

Automatic builds

Discovers .c files, resolves include paths from each package, then compiles and links in one command.

🐛

Debug & release modes

-g -DDEBUG by default, -O2 -DNDEBUG with --release — into separate build dirs.

🔒

Lock file

goose.lock pins exact commit SHAs for reproducible builds; verified on every build.

🏃

Task runner

Define named shell commands in goose.yaml and run them with goose task — unknown verbs fall back to tasks.

✅

Test runner

Drop .c files in tests/. Each is compiled and run independently — exit 0 means pass.

🔌

Plugin system

Custom transpilers for .l, .y, or any extension run before compilation into build/gen/.

🔄

CMake conversion

Auto-converts CMakeLists.txt packages to goose.yaml when fetched — no manual porting.

⬇️

Install command

Build a release binary and install it to your system with goose install and an optional prefix.

🧱

Library mode

libgoose.a is a generic framework — build your own Cargo-like tool for any language via callbacks.

How it works

A generic framework with C as the first plugin.

  • ✦One manifest — goose.yaml describes the project, build flags, dependencies, plugins, and tasks.
  • ✦Language-agnostic core — the framework knows nothing about C; it dispatches commands and resolves deps.
  • ✦Consumer plugins — the C toolchain is wired up through a GooseFramework with build/test/run callbacks.
  • ✦Reproducible — goose.lock records commit SHAs; builds check HEAD against the lock.
  • ✦Zero system deps — libyaml is vendored and compiled from source; nothing to install first.
goose.yaml │ goose <verb> (CLI dispatch) │ framework core (libgoose.a) │ ┌───────────┴───────────┐ resolve deps C consumer git · path · lock cc · cflags · link │ build/debug/<app>
Get started

Up and running in a minute.

Install with Homebrew (or the shell script), scaffold a project, and build.

terminal 
# 1. install (macOS, Homebrew)
brew tap wess/goose
brew install goose

# or install via the shell script
curl -fsSL https://raw.githubusercontent.com/wess/goose/main/install.sh | sh

# 2. create a new project
goose new myapp
cd myapp

# 3. build and run
goose run   # → Hello from myapp!
add a dependency 
# add a git dependency (optionally pin a version)
goose add https://github.com/user/mylib.git --version v1.0.0

# build with the lock file, then run the tests
goose build
goose test
